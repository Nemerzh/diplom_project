# Веб-інтерфейс (dashboard)

Односторінковий застосунок на **React 18** з **Vite 5**, маршрутизація **react-router-dom v6**, HTTP-клієнт **axios**. Збирається в статичні файли й роздається через **nginx** у Docker (порт контейнера **80**, у compose зазвичай **8081:80**).

## Навіщо він у стенді

- Візуалізація стану обліку: об’єкти, лічильники, покази, звіти, алерти.
- Адмін-розділ для реєстру підприємств, майданчиків, лічильників, сітки та правил сповіщень.
- Топологія мережі з **React Flow** (`@xyflow/react`) та допоміжні представлення (граф / дерево).

## Архітектура UI

| Шлях | Призначення |
|------|-------------|
| `/` | Дашборд (`DashboardPage`). |
| `/sites`, `/meters`, `/readings` | Об’єкти, лічильники, покази. |
| `/topology`, `/network` | Топологія та мережевий огляд. |
| `/reports`, `/alerts`, `/system` | Звіти, сповіщення, статус системи. |
| `/admin/*` | Адміністрування: enterprises, sites, meters, grid, alert-rules. |

Кореневі layout-и: **`Layout.jsx`** (основний UI), **`AdminLayout.jsx`** (адмінка).

Залежності для графіків і віджетів: **recharts**; для розкладки графа топології — **dagre** + **@xyflow/react**.

## Зв’язок з API

Усі запити йдуть на бекенд за базовим URL:

- **Локальна збірка:** змінна середовища **`VITE_API_BASE_URL`** на етапі `npm run build` (вшивається в бандл).
- **Docker:** у `Dockerfile` передається **`ARG VITE_API_BASE_URL`** (у compose для браузера користувача зазвичай **`http://localhost:8000`**).

У коді сторінок використовується цей префікс для axios-запитів до REST API бекенду.

## Команди розробки

| Команда | Дія |
|---------|-----|
| `npm install` | Встановити залежності. |
| `npm run dev` | Dev-сервер Vite на **5173** (`--host 0.0.0.0`). |
| `npm run build` | Production-збірка в **`dist/`**. |
| `npm run preview` | Перегляд зібраного бандла локально. |

## Зв’язок із деплоєм

- **Docker:** багатоетапний образ — build Node, потім nginx з **`nginx.conf`** і копією **`dist/`**.
- CORS на бекенді має дозволяти origin фронту (наприклад `http://localhost:8081`).

Коротко: **фронтенд — лише клієнт до бекенду**: він не зберігає «джерело правди», а відображає та редагує дані через API.
